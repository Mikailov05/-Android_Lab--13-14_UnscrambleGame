# Unscramble Game

## 1. Название проекта
Unscramble Game

## 2. Краткое описание игры
Игра, в которой игроку показывается слово с перемешанными буквами. Задача игрока — угадать исходное слово. За каждый правильный ответ начисляются очки. Игра состоит из 10 слов. После завершения игры показывается итоговый счёт и предлагается сыграть снова.

## 3. Технологии
- Kotlin
- Jetpack Compose
- ViewModel
- StateFlow
- Material Design 3

## 4. Функциональность приложения
- Отображение перемешанного слова
- Счётчик текущего слова (например, "Слово 1 из 10")
- Отображение текущего счёта
- Поле ввода ответа
- Кнопка "Проверить" для проверки ответа
- Кнопка "Пропустить" для перехода к следующему слову
- Сообщение об ошибке при неправильном ответе
- Диалог с результатами после завершения игры
- Кнопка "Играть снова" для перезапуска игры
- Сохранение состояния при повороте экрана

## Контрольные вопросы

### 1. Что такое ViewModel и зачем он нужен?
ViewModel — это компонент Android Jetpack, который хранит данные, связанные с UI, и автоматически переживает изменения конфигурации (поворот экрана, смена темы).

**Какую проблему решает ViewModel?**
При повороте экрана Activity уничтожается и создаётся заново, поэтому данные теряются. ViewModel остаётся в памяти, сохраняя данные.

**Что происходит с данными при повороте экрана без ViewModel?**
Данные сбрасываются к начальным значениям, прогресс игры теряется.

### 2. В чём разница между MutableStateFlow и StateFlow?
- **MutableStateFlow** — изменяемый поток данных (можно обновлять значение)
- **StateFlow** — неизменяемый поток только для чтения

**Зачем создавать два поля: _uiState и uiState?**
Приватное поле `_uiState` (MutableStateFlow) доступно только внутри ViewModel для изменения. Публичное поле `uiState` (StateFlow) доступно для чтения из UI.

**Что такое "backing property pattern"?**
Паттерн, где для одного свойства создаётся два поля: одно приватное изменяемое, другое публичное неизменяемое.

### 3. Почему в ViewModel нельзя хранить Context или View?
ViewModel живёт дольше Activity. Хранение Context или View может привести к утечке памяти.

**Что может произойти, если сохранить ссылку на Activity в ViewModel?**
Activity не сможет быть удалена сборщиком мусора, что приведёт к утечке памяти.

### 4. Чем StateFlow отличается от remember и rememberSaveable?
- **remember** — хранит состояние только во время жизни Composable
- **rememberSaveable** — хранит состояние в Bundle, переживает поворот экрана
- **StateFlow** — хранит состояние в ViewModel, переживает поворот экрана и поддерживает реактивность

**Когда лучше использовать каждый подход?**
- `remember` — для простых локальных состояний UI
- `rememberSaveable` — для данных, которые должны пережить поворот
- `StateFlow` — для бизнес-логики и данных, требующих реактивности

### 5. Что такое data class в Kotlin и зачем он используется для UI State?
**data class** — класс, предназначенный для хранения данных. Компилятор автоматически генерирует методы `equals()`, `hashCode()`, `toString()`, `copy()`.

**Какие методы автоматически генерируются для data class?**
`equals()`, `hashCode()`, `toString()`, `copy()`, `componentN()`.

**Зачем нужна функция copy()?**
Для создания копии объекта с изменением некоторых полей, не меняя исходный объект.

### 6. Как работает collectAsState() в Compose?
`collectAsState()` подписывается на StateFlow и преобразует его в State. При каждом изменении значения в StateFlow Compose автоматически перерисовывает UI.

**Почему UI автоматически обновляется при изменении StateFlow?**
`collectAsState()` создаёт наблюдаемое состояние. Когда значение меняется, Compose выполняет рекомпозицию.

### 7. Зачем разделять код на пакеты data и ui?
- **data** — для моделей данных и источников данных
- **ui (ui_model)** — для UI-компонентов и ViewModel

**Какие принципы архитектуры это поддерживает?**
Разделение ответственности (Separation of Concerns), чистая архитектура, удобство тестирования и поддержки.

### 8. Что происходит с ViewModel, когда приложение полностью закрывается (process death)?
ViewModel уничтожается, данные теряются.

**Сохраняются ли данные в этом случае?**
Нет, ViewModel не сохраняется при полном закрытии процесса.

**Какие альтернативы есть для долговременного хранения?**
- SavedStateHandle (для временного сохранения)
- DataStore / SharedPreferences
- База данных (Room)
- Файловое хранилище
